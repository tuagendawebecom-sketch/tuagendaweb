2. Why it was created this way
- The architecture is static-friendly and deploy-ready for Vercel.
- Commercial copy, demo data, WhatsApp messages, social links and tracking event names live in one editable file.
- Missing screenshots do not block launch: mockup components show clear placeholders until assets are replaced.
- TuAgendaWeb is positioned as a custom professional web solution, not as a generic SaaS.

11. Responsive and hydration fixes applied on 2026-05-16
- The reported hydration warning pointed at an extra html attribute: data-it-installed="true".
- The codebase did not contain SSR-unsafe render logic such as Date.now(), Math.random(), new Date(), localStorage, document reads, or typeof window branches used during render.
- data-it-installed is consistent with a browser extension injecting an attribute into the html element before React hydrates.
- Added suppressHydrationWarning only on the html element in src/app/layout.tsx so extension-injected html attributes do not create a false-positive warning. No dynamic attributes are added to html or body from client code.
- Browser-dependent tracking remains inside event handlers or Next Script execution after hydration.
- If the warning appears again in a normal browser, test in incognito or with extensions disabled. If it disappears there, the cause is the extension, not the app.

15. Optional demo screenshot capture
- Start the app first:
  npm run start -- --port 3001
- Then run:
  npm run capture:demos
- The script visits every /demos/[slug] route and saves screenshots into public/assets/demos/.
- Generated files:
  [slug]-desktop.png at viewport 1440x1000.
  [slug]-mobile.png at viewport 390x844.
  [slug]-cover.png at viewport 1200x750.
- Default base URL is http://localhost:3001.
- To use another URL:
  CAPTURE_BASE_URL=https://your-domain.com npm run capture:demos
- These generated files replace the placeholders automatically because the app already reads the same paths.
